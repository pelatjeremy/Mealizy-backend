# Guide beta testeur Mealizy

Merci de participer au test de Mealizy. Cette beta sert a verifier que l'application est claire, utile et stable dans des situations proches du quotidien.

## Objectif du test

Nous voulons comprendre:

- si les parcours principaux sont faciles a utiliser;
- si les suggestions de recettes sont pertinentes;
- si le planning et la liste de courses aident vraiment;
- ou l'application bloque, surprend ou manque de clarte.

Il ne s'agit pas de tout tester parfaitement. Le plus utile est de noter ce qui vous semble naturel, confus, lent ou casse.

## Comment se connecter

1. Ouvrez le lien de l'application fourni par l'equipe Mealizy.
2. Creez un compte avec une adresse email de test.
3. Connectez-vous avec cet email et votre mot de passe.
4. Si un compte test vous a ete fourni, utilisez directement ces identifiants.

Ne mettez pas de mot de passe que vous utilisez deja ailleurs.

## Quoi tester

Essayez de faire un scenario complet:

1. Creer un compte.
2. Ajouter quelques ingredients dans l'inventaire.
3. Chercher une recette.
4. Ouvrir une fiche recette.
5. Regarder les suggestions intelligentes.
6. Ajouter une recette au planning.
7. Generer une liste de courses.
8. Cocher quelques articles.
9. Modifier quelques preferences dans le profil.
10. Refaire une partie du parcours sur mobile si possible.

Vous pouvez aussi tester librement comme si vous prepariez vos repas de la semaine.

## Comment remonter un bug

Utilisez le modele `BUG_REPORT_TEMPLATE.md` ou envoyez les informations suivantes:

- ce que vous essayiez de faire;
- la page concernee;
- les etapes pour reproduire le probleme;
- ce que vous attendiez;
- ce qui s'est passe;
- une capture d'ecran si possible;
- votre appareil et navigateur.

Un bug utile n'a pas besoin d'etre long. Une phrase claire avec une capture d'ecran peut suffire.

## Informations importantes a fournir

Quand vous signalez un probleme, ajoutez si possible:

- ordinateur ou telephone;
- navigateur utilise;
- date et heure approximative;
- compte de test utilise, sans indiquer le mot de passe;
- message d'erreur affiche;
- recette, ingredient ou semaine concernee.

## Ce qui est normal en beta

Pendant cette beta controlee, il est normal que:

- certains textes soient encore perfectibles;
- quelques donnees de recettes soient incompletes;
- Spoonacular ou l'import externe ne soit pas toujours disponible;
- le design mobile ait encore besoin d'ajustements;
- certaines fonctionnalites avancees ne soient pas encore presentes.

En revanche, signalez toujours:

- une page bloquee;
- une action impossible a terminer;
- une erreur qui empeche de continuer;
- des donnees qui semblent perdues ou incoherentes;
- un parcours que vous ne comprenez pas.
